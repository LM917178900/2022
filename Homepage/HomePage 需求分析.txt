区域【New Product Launch Roadmap】

产品阶段及该阶段有多少产品
1、只对Ongoing产品有效，（如何判断产品的状态、字段【ODM Name】请参见通用规则中的第1条）。
●ODM Products X：展示ODM产品的数量，字段【ODM】值为非【N/A】时，这些产品便属于ODM类。
●Inhouse Products X：展示ODM产品的数量，字段【ODM】值为【N/A】时，这些产品便属于Inhouse类。
【OK2Ship】：该产品对应的【OK2Ship】取出来展示。（如何判断产品的状态、字段【ODM Name】请参见通用规则中的第3条）



产品阶段明细
1、与【产品阶段】有连动，默认展示第一个产品阶段的产品
2、当一页展示不完所有的产品时，鼠标移上去有滚动条。滚完这个阶段的产品，再滚就是下个阶段的了。
3、当前时间的不同，哪些是在同一个阶段，那个目标位置放的地方都不太一样。
4、取值时，同1个字段多个时取最小。

1、页面上展示的阶段分别是：POR、EVT、DVT、PVT、Ramp、Smart Launch。
2、Data Center→Product Schedule & TA→Tab【Product Schedule】中，Factory Type为非Oversea EMS的工厂，且Base为Y，的EVT Start 、DVT1 Start、PVT Start、OK2Pilot、OK2Ship、Sle的Actual值。
●POR：EVT Start 、DVT1 Start、PVT Start没有值，或 当前时间 <= EVT Start 。
●EVT：EVT Start  < 当前时间 <= DVT1 Start。
●DVT：DVT1 Start  < 当前时间 <= PVT Start。
●PVT：PVT Start < 当前时间 <= OK2Pilot。
●Ramp：OK2Pilot < 当前时间 <= OK2Ship。
●Smart Launch：OK2Ship < 当前时间 <= Sle。
3、该产品最新的风险状态需要去Master Data Config → Product Warning Light表里取，末尾默认展示最X种状态中风险最低的那个，悬浮才展示全部的风险状态。



悬浮提示
1、带条件，查该产品最新版的数据。
2、没有对应的菜单权限，点了是跳不进去。
●Schedule：点击带条件新开页面到Dashboard→Schedule Dashboard
●Biz Case：点击带条件新开页面到Data Center→Business Case→Tab【Summary-Moto】或【Summary-Gaming Phone】
●PDP：点击带条件新开页面到Data Center→PDP→Tab【Region】
●LTF：点击带条件新开页面到Data Center→LTF
●OD：点击带条件新开页面到Data Center→Product OD & Picture & CMF
●CMF：点击带条件新开页面到Data Center→Product OD & Picture & CMF→下该产品在附件中最新上传的PDF

 
 
 
 

区域【New Product Supply Dashboard】【同 之前的home page 】

统计图
1、PSS：折线，Ongoing所有产品的最新发版的PSS里的【PSS (CN/BR/IN FG+AR SKD)】
2、PDP：折线，Ongoing所有产品的最新版PDP。
3、BC：柱子，Ongoing所有产品的，状态为DOS1的，最新的发版的PSS或最新归档的PSS里取最小的，的【PSS (CN/BR/IN FG+AR SKD)】

4、以月为单位，当前时间所在月，向后11个月，共12个月。

（如何判断产品的状态请参见通用规则中的第1条）
1、找到这个产品在Data Hub→Product Schedule下最新的Schedule，且把【Base】为【No】的数据过滤掉，剩下的数据进行下面的计算。
2、把里面所有非Oversea EMS工厂的【Milestone】为【CWV/Actual】的【Sle】取出来。

3、如果1个【CWV/Actual】都没值就取【Milestone】为非Oversea EMS工厂的【Plan】类型，其中【Sle】值最小的那个。

4、而取出来的这个【Sle】所在数据的【Design Type】、【Product Status】、及这个【Sle】所对应的财年【Fiscal Year】便是这个产品对应的Design Type、Product Status、Fiscal Year。







区域【New Product Launch Qty by Site】【季度分组，每个厂的产品数量，中国只算一个厂，同一个产品只能落在一个季度，取最小的季度】
季度产品统计

1、当前时间所在财季向后3个Q，总共4个Q。需要把Q简写，如FY2021-Q1，需要简写为FY21-Q1。
2、环形饼图：Data Hub→Product Schedule中每个产品最新的Schedule数据中，
按工厂分组（Master Data Config → Factory List中Type为非Oversea EMS的都是CN类，EMS的是啥就啥类），Milestone→Plan→字段OK2Ship的时间落在哪个Q，这个产品就是在哪个Q。（可能CN有多条Plan的，就取最早的那个。）----不分产品状态，只要落在这个Q都要记
3、假设某个产品A，过滤了里面有4个工厂WH的OK2Ship在FY21-Q1、BR的OK2Ship在FY21-Q1、IN的OK2Ship在FY21-Q2、TINNO的OK2Ship在FY21-Q2，且其中TINNO、WH的都应该归到CN这类，那这时在FY21-Q1的CN的记数就要为1，BR的记数就要为1。FY21-Q2关于IN就要记1，而TINNO就不记。（TINNO在Q1也不记）
4、圆心的数据：等于CN的数。
5、CN、AR、BR、IN永远都是这4个写死放那里，如果没有数就是0。
